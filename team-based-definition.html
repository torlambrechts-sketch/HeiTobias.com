Integration with other modules

This module is the single canonical writer of role versions. Every other module reads downstream. The integration contracts here are tested explicitly so the system doesn't silently drift toward "latest by family" instead of pinned versions, or read peer-personality data where none should exist.

Role Profile detail page

Renders the signed-off version with full provenance (ICC, Kendall's W, evaluator count, reconciliation summary, attribution).
→ /roles/:role_id/:version

Requisitions & hiring

A requisition pins to role_version_id. A superseding version does NOT change the requisition's frozen view — fairness across candidates depends on it.
→ requisitions.role_version_id (immutable)

Fit scoring

Scores a candidate against the version pinned at decision time, never "latest." Captured in decision_artefact.
→ fit_score.against_role_version_id

Frameworks Library

Seeds the initial draft (competencies, family-default trait bands, context defaults). Evaluators tune; they never start blank.
→ frameworks.role_family_templates

Re-fit engine (Phase 3)

Reads evolved versions to compute fit-trajectory. Evolution vector generated at reconciliation labelled as forecast.
→ refit.against_role_version_id (time-series)

Team-gap context

Reads role version + members' OWN profiles, never peer-personality. Surveillance guardrail enforced at schema level.
→ team_gap.{role_version_id, members_own_profiles}

Phase 4 audit / DPIA export

Includes the full Delphi provenance per role version: evaluators, submissions, divergence, reconciliations, sign-off attribution.
→ exports.ai_act_annex_iv · exports.dpia_art_35

Decision artefacts

Every reconciliation override + sign-off writes a decision_artefact (Hardening §A4). The role version carries pointers to all of them.
→ decision_artefacts.linked_role_version_id
